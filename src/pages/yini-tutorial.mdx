# Quick Tutorial: YINI Basics

YINI makes configs readable for humans *and* consistent for machines ‚Äî like INI, but with **real structure**, arrays, objects, and comments.

A quick tutorial with the basics of the YINI format and how to validate it.

## 1) What is YINI (in one line)?
A human-friendly config format (like INI) **with real structure**: sections, clear nested sections, arrays, objects, and unambiguous, predictable rules.

---

## 2) Create your first file

Make a file named **`config3.yini`** and paste:

```yini
^ App
name = "Demo"
version = "1.0.0"
features = ["search", "dark-mode"]  // comments are allowed

^ Server
host = "0.0.0.0"
port = 8080

^^ Logging
enabled = false
```

### üí° What's happening
- `^` starts a section. More carets = nested section (`^^ Logging` is inside `Server`).
- `key = value` defines a setting.
- Arrays (lists): `[1, 2, 3]`, objects: `{ key: value }`.
- Comments: `//` or `#` (include a space after `#` for inline comments).
- Want more of YINI's syntax and structure? Read [Intro to YINI format](intro-yini-config-format).

---

## 3) Validate your config (in CLI)
In your terminal (from the same folder as your file), run:
```bash
npx yini-cli validate ./config3.yini
```

You should see something like:

```bash
‚úÖ Validation passed ‚Äî 0 errors, 0 warnings.
```

If YINI reports 0 errors, 0 warnings, everything's fine.
A non-zero exit code means something went wrong.

### More commands and options 
To explore more commands and options, run:
```bash
npx yini-cli --help
```

## 5) npm script
Use this in **CI** to catch config mistakes early:

```js
{
  "scripts": {
    "config:check": "npx yini-cli validate ./config3.yini"
  }
}
```

Run with:
```bash
npm run config:check
```

### ‚ö†Ô∏è Common Questions 

#### Does indentation matter? 
No ‚Äî indentation is only for readability.  
Structure is determined by section markers (`^`, `^^`, etc.), not spaces.

#### Can I mix comment styles?
Yes ‚Äî YINI supports both `//` and `#` for inline comments.  
For consistency, use just one style per file.
